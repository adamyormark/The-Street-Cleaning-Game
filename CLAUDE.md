# The Street Cleaning Game

A single-page web app for managing alternate side parking in Fort Greene, Brooklyn.

## Architecture

- **Single `index.html`** — vanilla HTML/CSS/JS, no dependencies, no build step at serve time
- **Data**:
  - Hardcoded 2026 NYC ASP suspension calendar from NYC DOT
  - Inlined `SPOTS` database: ~1,800 block-side cleaning windows within ~1.5 mi of home, generated from NYC OpenData (see below)
- **Storage**: `localStorage` for user preferences (`settings.mySpots`, `settings.blockedEvenings`)
- **Hosting**: GitHub Pages (push to `main` to deploy)

## ASP Calendar

The 2026 suspension dates are hardcoded in `index.html`. Source: [NYC DOT ASP Calendar](https://www.nyc.gov/html/dot/downloads/pdf/asp-calendar-2026.pdf). Update annually.

## Spots Database

The inline `STREETS` + `SPOTS` block in `index.html` (between the `SPOTS-DATA-START` / `SPOTS-DATA-END` markers) is generated by `scripts/build-spots.py`. It queries [NYC OpenData `nfid-uabd`](https://data.cityofnewyork.us/Transportation/Parking-Regulation-Locations-and-Signs/nfid-uabd) (no auth needed), filters by sign coordinates to a 7,920 ft radius (≈1.5 mi, ≈8-min bike) around 253 Cumberland St, and emits compact rows of the form `[streetIdx, fromIdx, toIdx, side, dayMask, startMin, endMin, dxFromHome, dyFromHome]`.

Regenerate when ASP signs change (yearly is plenty):

```sh
python3 scripts/build-spots.py
```

The script edits `index.html` in place between the markers — no need to copy/paste. Spots are pre-sorted by distance from home so the migration heuristic (first match wins) seeds defaults that are actually nearby.

## User Context

- Home: 253 Cumberland St, Brooklyn 11205 (between Lafayette Ave and DeKalb Ave, west side)
- Works M-F, leaves 8:30 AM, returns 5:15 PM
- Cannot move car during cleaning hours (always during work hours), so the rec engine assumes "move the evening before"
